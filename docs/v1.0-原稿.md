> **产品定位**:AI 驱动的品牌独立站生成平台。用户粘贴现有店铺 URL 或描述需求,AI 自动抓取商品、重构品牌视觉、生成带 3D 动效的艺术级独立站,托管在 Forgely 平台,3-5 分钟上线欧美市场。

> **核心差异化**:美学 + 交互 + 3D 效果,对标 BIOLOGICA、Recess、Magic Spoon、Aesop 这类顶级 DTC 品牌站的质感。
>
> **Slogan 候选**:
> - "Forge your brand, not just a store"
> - "Your brand, forged by AI"
> - "From link to launch in 5 minutes"

## 1. 产品定义与核心决策

### 1.1 目标用户

两类核心用户,一套产品服务:

**Persona A:中国工厂品牌主(出海型)**
- 画像:30-50 岁,有工厂或供应链,想做自己的品牌卖欧美
- 痛点:英文差、不懂欧美审美、不会前端、不懂 SEO
- 需求:中文界面、强 AI 自动化、傻瓜式上手、跨境合规指引
- 付费意愿:高($99-299/月能接受)

**Persona C:欧美小 DTC 品牌主**
- 画像:25-40 岁,有品牌想法,可能已在 Shopify
- 痛点:Shopify 主题同质化、请设计师贵、想要高级感
- 需求:英文界面、审美高级、支持代码导出、高定制自由度
- 付费意愿:中高($49-149/月)

### 1.2 核心决策摘要

| 决策项 | 选择 | 理由 |
|---|---|---|
| 商业模式 | 托管 SaaS + 代码导出(溢价) | 留存好、现金流稳、IP 保护 |
| 电商后端 | Medusa.js(自建) | 控数据、无抽成、差异化 |
| 前端技术栈 | Next.js 14 + R3F + Tailwind | 生态成熟、SSR 友好 |
| 部署平台 | Cloudflare Pages + Workers | 成本低、出站零费、边缘快 |
| 多租户策略 | 共享 Medusa(MVP)→ 按 plan 独立实例(扩展) | MVP 快速验证 |
| 多语言 | MVP 只英文 → 后续加 DE/FR/ES/PT | 复杂度可控 |
| 代码导出 | 快照模式(带走代码,不带走 AI 能力) | 保护核心 IP |
| 支付 | Stripe + PayPal + Crypto + 插件化 | 覆盖欧美主流 |
| 目标市场 | 欧美(US + EU + UK + CA) | 购买力强、DTC 文化成熟 |
| 核心壁垒 | 美学 + 交互 + 3D 组件库深度 | 软壁垒需持续投入 |

### 1.3 定价计划

| 计划 | 价格 | 额度 | 功能 |
|---|---|---|---|
| Free | $0 | 注册送 1 次完整生成 | 带水印、仅预览、不能绑域名 |
| Starter | $49/月 | 3 个站、无限生成 | 基础模板、去水印、自定义域名 |
| Pro | $149/月 | 10 个站、无限生成 | 全模板、3D 效果、代码导出、AI 编辑 |
| Agency | $499/月 | 50 个站、白标 | 客户管理、多人协作、优先支持 |

前 3-6 个月全试用免费,收集数据和反馈。

## 3. AI 责任分工与编排架构

这是整个系统的灵魂。我把生成过程拆成 **7 个独立的 AI Agent**,每个 Agent 职责单一、可独立测试、可独立优化。

### 3.1 Agent 总览

```
┌──────────────────────────────────────────────────────────────┐
│                    编排总管(LangGraph)                     │
│              控制状态流转、重试、降级、并行                   │
└──────────────────────────────────────────────────────────────┘
    │         │         │         │         │         │         │
    ▼         ▼         ▼         ▼         ▼         ▼         ▼
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│Agent1│ │Agent2│ │Agent3│ │Agent4│ │Agent5│ │Agent6│ │Agent7│
│爬虫  │ │分析师│ │策划师│ │文案员│ │艺术家│ │工程师│ │质检员│
└──────┘ └──────┘ └──────┘ └──────┘ └──────┘ └──────┘ └──────┘
```

### 3.2 每个 Agent 的详细职责

#### Agent 1:Scraper(爬虫)
**职责**:从源站提取所有原始数据
**模型**:Claude Sonnet(分析 DOM)+ Playwright 执行
**输入**:源站 URL
**输出**:
```json
{
  "products": [{ "title","price","images","description","variants"... }],
  "siteMetadata": { "title","description","keywords","schema","og"... },
  "screenshots": { "homepage","productPage","collection" },
  "assets": { "logo","fonts","colors" },
  "seo": { "sitemap","robots","internalLinks" }
}
```
**关键设计**:
- 优先使用各平台官方 API(Shopify `/products.json`、WooCommerce REST API)
- 失败降级到 Playwright DOM 解析
- 反爬站点接入 ScraperAPI / Bright Data
- 所有抓取结果缓存 24h(避免重复抓取)

#### Agent 2:Analyzer(风格分析师)
**职责**:理解源站的品牌基因
**模型**:Claude Sonnet + Vision
**输入**:截图 + 原始数据
**输出**:`StyleDNA` 结构化对象
```json
{
  "mood": "luxury_wellness | minimal_tech | playful_youth | ...",
  "colors": { "primary","secondary","accent","bg","fg" },
  "typography": { "headingFont","bodyFont","scale" },
  "imageStyle": "product_floating | flat_editorial | lifestyle_hero",
  "copyTone": "confident_clinical | warm_friendly | bold_rebellious",
  "category": "supplements | fashion | beauty | food | tech | home",
  "targetDemographic": { "age","gender","income","psychographic" },
  "competitorRef": ["biologica.com","recess.com"]
}
```

#### Agent 3:Planner(策划师)
**职责**:规划新站的信息架构 + 关键词策略
**模型**:Claude Opus(复杂推理)
**输入**:StyleDNA + 商品列表 + 目标市场
**输出**:完整 `SiteDSL`
```json
{
  "pages": [
    {
      "path": "/",
      "type": "Homepage",
      "blocks": [
        { "component": "HeroVideoBackground", "props": {...} },
        { "component": "ProductShowcase3D", "props": {...} },
        ...
      ],
      "seo": { "title","description","h1","keywords","schema" },
      "geo": { "intent","aiQueryTargets","citableFacts" }
    },
    ...
  ],
  "theme": { colors, fonts, motionLevel },
  "seoStrategy": {
    "primaryKeywords": [...],
    "longTailKeywords": [...],
    "internalLinkMap": {...}
  },
  "geoStrategy": {
    "llmsTxt": true,
    "faqPages": [...],
    "authorityBuilders": [...]
  }
}
```

#### Agent 4:Copywriter(文案员)
**职责**:为每个页面、每个商品生成欧美 DTC 风格文案
**模型**:Claude Sonnet(流式输出)
**输入**:SiteDSL + StyleDNA + 标杆品牌文案库(few-shot)
**输出**:每个文案 block 的英文内容
**特殊要求**:
- 避免中式英语
- 学习 DTC 文案模板(下面会列)
- 关键词密度符合 SEO 要求
- 包含"AI 可引用"的事实陈述(为 GEO)
- 每个 product 必须带 FAQ(最少 3 条)

#### Agent 5:Artist(艺术家)
**职责**:生成所有视觉素材
**模型**:多模型协作
- Ideogram 3.0 / Recraft V3 → Logo
- Flux 1.1 Pro → 场景图、Lifestyle 图
- Flux Kontext → 保持产品一致性
- Kling 2.0 / Runway Gen-4 → Hero 视频
- Meshy / Tripo → 产品 3D 模型
- remove.bg → 抠图
- gltf-transform → 3D 模型压缩
**输出**:所有素材的 R2 URL
**关键设计**:
- 并行生成(3-5 个素材同时跑)
- 每类素材有 3 个候选 prompt,自动选最佳
- 失败降级(视频生成失败→用静态图)

#### Agent 6:Engineer(工程师)
**职责**:把 SiteDSL 编译成可部署代码
**模型**:不用 AI,**纯确定性编译器**(关键!)
**输入**:完整 SiteDSL + 所有文案 + 所有素材
**输出**:
- `next-app/`:完整 Next.js 项目
- Medusa seed 数据(商品 / 分类 / 支付配置)
- SEO 资产(sitemap.xml / robots.txt / llms.txt)
- 合规页面(隐私政策 / TOS / DSA)

**为什么不用 AI**:这一步必须 100% 可预测、可测试、可回归。AI 写 React 代码错误率 5-10%,不能要。

#### Agent 7:Compliance Officer(合规官)
**职责**:审查生成内容的合规性
**模型**:Claude Sonnet(专项训练的 prompt)
**检查维度**:
- FDA 禁用词(保健品禁 "cure/treat/prevent disease")
- FTC 广告法(禁 "#1"、虚假证言)
- 商标侵权
- 虚假评论标注
- 医疗/金融免责声明
- GDPR/CCPA/DSA 必备页面
**输出**:合规报告(红/黄/绿三级)
**流程**:红色→必改,黄色→用户确认,绿色→放行

### 3.3 Agent 之间的协作模式

使用 **LangGraph** 做状态机编排:

```typescript
// 伪代码
const workflow = new StateGraph({
  channels: {
    sourceUrl: null,
    scrapedData: null,
    styleDNA: null,
    siteDSL: null,
    copy: null,
    assets: null,
    code: null,
    complianceReport: null,
    deploymentUrl: null,
  }
})

workflow.addNode("scraper", scraperAgent)
workflow.addNode("analyzer", analyzerAgent)
workflow.addNode("planner", plannerAgent)
workflow.addNode("copywriter", copywriterAgent)
workflow.addNode("artist", artistAgent)
workflow.addNode("engineer", engineerAgent)
workflow.addNode("compliance", complianceAgent)
workflow.addNode("deployer", deployerAgent)

// 串行链路
workflow.addEdge("scraper", "analyzer")
workflow.addEdge("analyzer", "planner")

// 并行链路(文案和素材可以同时生成)
workflow.addParallelEdges("planner", ["copywriter", "artist"])
workflow.addEdge(["copywriter", "artist"], "engineer")

// 合规检查 + 条件分支
workflow.addEdge("engineer", "compliance")
workflow.addConditionalEdge("compliance", (state) => {
  if (state.complianceReport.hasRed) return "copywriter" // 回到文案重写
  return "deployer"
})
```

### 3.4 失败恢复设计

每个 Agent 完成后在 Postgres 存 checkpoint:
```sql
CREATE TABLE generation_checkpoints (
  generation_id UUID,
  step TEXT,  -- 'scraping' | 'analyzing' | ...
  status TEXT, -- 'done' | 'failed' | 'retrying'
  output JSONB,
  created_at TIMESTAMP
)
```

失败时从最后一个 `done` 的 checkpoint 继续,不从头开始。

## 6. SEO + GEO 内化到生成流程

### 6.1 原则

SEO/GEO 不是"生成后优化",而是**从 DSL 生成的第一步就是必填字段**。编译器看到缺失直接失败。

### 6.2 每个页面必填的 SEO 字段

```typescript
interface PageSEO {
  title: string              // 50-60 字符
  description: string        // 150-160 字符
  canonical: string
  h1: string                 // 每页有且仅有 1 个
  targetKeywords: string[]   // 主词 + 长尾
  keywordDensity: {          // 自动计算,用于校验
    [keyword: string]: number
  }
  ogImage: string            // 1200x630
  ogType: 'website' | 'product' | 'article'
  twitterCard: 'summary_large_image'
  hreflang: { lang: string, url: string }[]
  robots: string             // 'index,follow' 等
}
```

### 6.3 每个页面必填的 GEO 字段

```typescript
interface PageGEO {
  intent: string             // 用户意图分类
  aiQueryTargets: string[]   // 期望被 AI 引用的查询
  citableFacts: {            // AI 友好的事实陈述
    fact: string
    evidence?: string
  }[]
  faqSchema: boolean
  reviewSchema: boolean
  author?: {                 // 权威署名
    name: string
    credentials: string
    bio: string
  }
  topicalAuthority: string[] // 覆盖的主题
}
```

### 6.5 关键词研究(Planner Agent 用)

**接入 DataForSEO API**:
```typescript
// Planner 调用
const keywords = await dataForSEO.keywordIdeas({
  seedKeyword: product.category,  // 如 "women vitamins"
  location: 'United States',
  language: 'en'
})

// 按搜索量 + 竞争度筛选
const targetKeywords = keywords
  .filter(k => k.searchVolume > 100 && k.competition < 0.7)
  .slice(0, 20)
```

策略:
- 1-2 个主词(品类 head keyword)
- 10-15 个长尾词(low competition + high intent)
- 每个产品页配 3-5 个长尾作为目标

### 6.6 内部链接自动化

Planner Agent 生成站点时就规划好内链:
- Hub & Spoke 结构:分类页(hub)→ 产品页(spoke)
- 相关产品推荐(每个产品页 4-6 个内链)
- 博客文章自然提及产品(带锚文本)

### 6.7 llms.txt + llms-full.txt 自动生成

**llms.txt 结构**(站点地图 + 关键页面):
```markdown
# Brand Name

> Brand description and value proposition

## Products
- [Product 1](url): short description
- [Product 2](url): short description

## About
- [Our Story](url): brand story summary
- [Ingredients](url): what we use and why

## FAQ
- [Frequently Asked Questions](url)
```

**llms-full.txt**(完整 markdown 内容,供 LLM 爬取训练或引用):
```markdown
# Brand Name

Full homepage content in markdown...

# Product 1

Full product page content in markdown...

# FAQ

Q: ...
A: ...
```

自动生成规则:
- 每次内容更新,重新编译
- 在 `/llms.txt` 和 `/llms-full.txt` 路由返回
- 在 robots.txt 明确允许

### 6.8 性能预算(Core Web Vitals)

每个生成的页面,编译时强制检查:
- LCP ≤ 2.5s(移动端 4G 模拟)
- INP ≤ 200ms
- CLS ≤ 0.1
- 初始 JS ≤ 200KB
- 初始图片总 ≤ 1MB

不过关 → 回到 Generator 降级方案:
- Hero 视频 → poster 图
- 3D 组件 → 2D 组件
- 高清图 → WebP/AVIF 压缩版

### 6.9 自动提交收录

生成部署后自动执行:
- **Google Indexing API**:提交所有 URL
- **IndexNow API**:提交给 Bing/Yandex
- **生成 sitemap.xml** 并 ping Google / Bing
- 24 小时后扫描收录情况

```typescript
// payment-provider-nowpayments.ts
export default class NOWPaymentsProvider extends AbstractPaymentProvider {
  async initiatePayment(context) {
    const payment = await nowpayments.createPayment({
      price_amount: context.amount,
      price_currency: context.currency,
      order_id: context.resource_id,
      ipn_callback_url: `${APP_URL}/webhooks/nowpayments`,
    })
    return { session_data: payment }
  }
  // ... capturePayment, refundPayment, etc.
}
```


## 15. 风险点与应对

### 15.1 技术风险

| 风险 | 影响 | 应对 |
|---|---|---|
| AI 生成不稳定 | 用户体验差 | 用 DSL + 确定性编译,AI 只填参数不写代码 |
| 视频/3D 生成慢 | 用户等待久 | 用占位符,并行生成,好了替换 |
| 爬虫被封 | 生成失败 | 多 IP 池、ScraperAPI 兜底、用户手动上传降级 |
| Medusa 扩展性 | 用户多了慢 | 共享 → 独立实例分层;加缓存 |
| 3D 性能差 | SEO 受影响 | 强制移动端降级、性能预算硬门槛 |

### 15.2 法律风险

| 风险 | 应对 |
|---|---|
| 用户侵权(参考别人的站) | 注册勾选 + TOS 免责 + 只提取风格不用素材 |
| FDA/FTC 违规 | 合规 Agent 强制审查 + 敏感品类黑名单 |
| GDPR 违规 | 自动生成 Privacy Policy + Cookie Banner |
| 商标侵权 | 合规 Agent 检查品牌名冲突 + USPTO/EUIPO 免费查询 |
| 支付合规 | Stripe 禁用品类黑名单 + MiCA 合规提示 |

### 15.3 商业风险

| 风险 | 应对 |
|---|---|
| Atlas 等竞品入侵 | 聚焦高端品牌美学 + 3D 深度,不拼 dropshipping 市场 |
| 用户留存低(生成完就走) | 托管 + 订阅 + 代码导出需付费 + 持续 AI 编辑能力 |
| API 成本失控 | 每用户生成配额 + 缓存常见结果 + 批量请求 |
| 支付率低 | 30 天免费试用 + 站做好了才付钱 + 分层定价 |

### 15.4 组织风险

| 风险 | 应对 |
|---|---|
| 没有前端工程师 | 70% 抄开源 + 30% 外包关键 3D 组件 + AI 编程工具辅助 |
| 审美品味不够 | 深度研究 150 站 + 聘请兼职设计顾问 + A/B 测试真实用户偏好 |
| 全靠 AI 开发质量不稳 | 严格的 Task 拆分 + 每 Task 人工 Review + 自动化测试 |
